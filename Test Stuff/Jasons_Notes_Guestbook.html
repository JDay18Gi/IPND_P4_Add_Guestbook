Lesson 1 The Basics of the Web and HTML


Web Basics

  • A collection of web pages
  • Computer makes requests to servers (via HTTP). Server responds with files.
  • Web invented in 1990
  • 30+ billion pages!
  • HTML = Hypertext Markup Language. It is the base for nearly all web pages. HTML docs are the majority of documents on the web.


HTML

  • Written like below, where 'name' is a tag. (But some tags do not need to be closed.)

<name> contents </name>

  • Some examples of tags are these:

<em> contents </em>

- for italics

<b> contents </b>

- for bold

<a href="http://www.example.com/"> Example Link </a>

- for Example Link

  • In web development, an "element" refers to the opening and closing tag along with the contents of the tag.

Inline and Block Elements

  • Inline elements have no box around them (think text without a textbox). Examples of inline elements are these: <a>, <span>, <br>, and <img>, <strong>

  • Block Elements have an invisible box around them (like a textbox).

Computers are Dumb

  • We say this with compassion, but computers need all these special tags and elements because they cannot understand things the way humans do. This is important for programmers to keep in mind because even little errors can cause big problems.


Lesson 2: Structure and HTML

All Webpages are Boxes

  • These boxes make up the layout of the webpage.
  • Even circular items have invisible boxes around them!
  • Each box has a few aspects to it: margin, border, padding, and the item itself.

DOM (Document Object Model)

  • DOM refers to the tree-like structure used to write code for a webpage.
  • Elements that are most subodinate (furthest to the right) are affected by the "parent" elements which are less indented.
  • Spans and Divs are used to group code. Span is inline. Div is block (remember that means it has an imaginary box around it).

Images

  • Images can be included by using the following code:
            <img src="image_url" alt="alt_text">
  • Alt text is important for the visually impaired and if the image doesn't load.

Lesson 3: Adding CSS Style

  • Avoid repetition in your code!!!

Adding CSS Style to HTML

  • CSS = cascading style sheets
  • Cascading = the most specific rule gets applied and rules and inherited from elements further to the left.
  • inheritance: text related properties are inehrited (from the left to the elements further to the right), box related properties are not inherited.
  • CSS ID Selectors: '.', '#', etc.

Box Sizing & Structure

  • use percentages
  • use "border-box"

Lesson 4: Intro to "Serious" Programming

  • A Computer is a universal machine that can be programmed to do almost aything.

Computer Programming

  • Computers can do almost anything but they need instructions. These instructions are the programs.
  • Computers use compilers or interpreters (programs that know hot to read different coding languages.)
  • Python is an interpreter and the Python language is interpreted to make sense to the computer so it knows what to do.
  • There are 5 ways to think like a programmer:

    • 1. Procedural Thinking

    2. Technological Empathy

    3. Abstract Thinking

    4. Debugging

    5. Systems Thinking


Grammar and Syntax

  • Computers need special languages because they cannot think for themselves to interpret things like humans do. They need a very exact language.
  • Computers use a form called "Backus-Naur Form"(BNF). This is very precise.
  • Computers use the form: <Non-terminal> = <replacement>
  • People used to think computers could only do math but they can do a LOT more than that!

Lesson 5: Variables and Strings

Variables

  • Variables can be assigned any meaning we want. They can be assigned the meaning of numbers, lists of numbers, formulas, words, phrases, even huge pages of text or code.
  • Variables conveniently allow long complex statements to be contained in just a word or letter.
  • Python is an interpreter and the Python language is interpreted to make sense to the computer so it knows what to do.


Strings

  • Strings are characters surrounded by quotes.
  • Strings can be added together but they cannot be added to numbers.
  • Two important features of strings is that they can be searched and they can be indexed.
  • x.find('a') will find the first occurance of 'a' in x.

Lesson 6: Funcitons

  • Functions take in inputs and create outputs
  • 'return' - (like the answer) what the function spits out
  • Functions have to be made (defined) before they can be used.
  • Calling a function means using the function.
  • Here is a function that multiples a list of numbers:

    • def product_list(n):

         x=0

        result=1

        while x<len(n):

            result=result*n[x]

            x=x+1

        return result


Lesson 7: Decisions and Repetition

  • Functions can use comparisons < and >
  • != means not equal
  • == means equal (like in math, not assignment like in Python)
  • if statements can be used in functions for example:

    • def bigger (a,b):

         if a>b:

            return a

        return b


Lesson 8: Structured Data

  • Lists are another form used in Python. They are written like so: p=[0,1,2,'a','aba',['a','b',1]]
  • Notice that lists can contain both letters or numbers. There can also be lists within the lists.
  • Lists, unlike strings, can be manipulated
  • Loops can be used for lists similar to how while is used for strings.
  • Here is an example:

    • def greatest(p):

         big=0

         for i in p:

         if i>big:

             big=i

         return big
