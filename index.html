Lesson 1 The Basics of the Web and HTML


Web Basics

  • A collection of web pages
  • Computer makes requests to servers (via HTTP). Server responds with files.
  • Web invented in 1990
  • 30+ billion pages!
  • HTML = Hypertext Markup Language. It is the base for nearly all web pages. HTML docs are the majority of documents on the web.


HTML

  • Written like below, where 'name' is a tag. (But some tags do not need to be closed.)

<name> contents </name>

  • Some examples of tags are these:

<em> contents </em>

- for italics

<b> contents </b>

- for bold

<a href="http://www.example.com/"> Example Link </a>

- for Example Link

  • In web development, an "element" refers to the opening and closing tag along with the contents of the tag.

Inline and Block Elements

  • Inline elements have no box around them (think text without a textbox). Examples of inline elements are these: <a>, <span>, <br>, and <img>, <strong>

  • Block Elements have an invisible box around them (like a textbox).

Computers are Dumb

  • We say this with compassion, but computers need all these special tags and elements because they cannot understand things the way humans do. This is important for programmers to keep in mind because even little errors can cause big problems.


Lesson 2: Structure and HTML

All Webpages are Boxes

  • These boxes make up the layout of the webpage.
  • Even circular items have invisible boxes around them!
  • Each box has a few aspects to it: margin, border, padding, and the item itself.

DOM (Document Object Model)

  • DOM refers to the tree-like structure used to write code for a webpage.
  • Elements that are most subodinate (furthest to the right) are affected by the "parent" elements which are less indented.
  • Spans and Divs are used to group code. Span is inline. Div is block (remember that means it has an imaginary box around it).

Images

  • Images can be included by using the following code:
            <img src="image_url" alt="alt_text">
  • Alt text is important for the visually impaired and if the image doesn't load.

Lesson 3: Adding CSS Style

  • Avoid repetition in your code!!!

Adding CSS Style to HTML

  • CSS = cascading style sheets
  • Cascading = the most specific rule gets applied and rules and inherited from elements further to the left.
  • inheritance: text related properties are inehrited (from the left to the elements further to the right), box related properties are not inherited.
  • CSS ID Selectors: '.', '#', etc.

Box Sizing & Structure

  • use percentages
  • use "border-box"

Lesson 4: Intro to "Serious" Programming

  • A Computer is a universal machine that can be programmed to do almost aything.

Computer Programming

  • Computers can do almost anything but they need instructions. These instructions are the programs.
  • Computers use compilers or interpreters (programs that know hot to read different coding languages.)
  • Python is an interpreter and the Python language is interpreted to make sense to the computer so it knows what to do.
  • There are 5 ways to think like a programmer:

1. Procedural Thinking

2. Technological Empathy

3. Abstract Thinking

4. Debugging

5. Systems Thinking


Grammar and Syntax

  • Computers need special languages because they cannot think for themselves to interpret things like humans do. They need a very exact language.
  • Computers use a form called "Backus-Naur Form"(BNF). This is very precise.
  • Computers use the form: <Non-terminal> = <replacement>
  • People used to think computers could only do math but they can do a LOT more than that!

Lesson 5: Variables and Strings

Variables

  • Variables can be assigned any meaning we want. They can be assigned the meaning of numbers, lists of numbers, formulas, words, phrases, even huge pages of text or code.
  • Variables conveniently allow long complex statements to be contained in just a word or letter.
  • Python is an interpreter and the Python language is interpreted to make sense to the computer so it knows what to do.

Strings

  • Strings are characters surrounded by quotes.
  • Strings can be added together but they cannot be added to numbers.
  • Two important features of strings is that they can be searched and they can be indexed.
  • x.find('a') will find the first occurance of 'a' in x.

Lesson 6: Funcitons

  • Functions take in inputs and create outputs
  • 'return' - (like the answer) what the function spits out
  • Functions have to be made (defined) before they can be used.
  • Calling a function means using the function.
  • Here is a function that multiples a list of numbers:

def product_list(n):

     x=0

    result=1

    while x<len(n):

        result=result*n[x]

        x=x+1

    return result

def bigger (a,b):

Lesson 8: Structured Data

  • Lists are another form used in Python. They are written like so: p=[0,1,2,'a','aba',['a','b',1]]
  • Notice that lists can contain both letters or numbers. There can also be lists within the lists.
  • Lists, unlike strings, can be manipulated
  • Loops can be used for lists similar to how while is used for strings.
  • Here is an example:

def greatest(p):

     big=0

     for i in p:

     if i>big:

         big=i

     return big


Lesson 9: Understanding OOP

  • Object-oriented programming is a form of programming based on the concepts of objects. In laymans terms, the OOP method is a method of using many smaller programs to work together to accomplish the overall talk of the program. Program tasks are broken down into small parts that are completed by functions and processes. These objects are often contained in classes (see below). The OOP method causes us to think about tasks in their parts. By breaking down tasks into their parts we can identify classes that apply to the procedures we need to accomplish and then create objects for those classes that are specific to our needs.
  • Vocabulary is a very important part of OOP.

Classes

Classes are like a blueprint for an object.
  • They can be used to make many different types of buildings but all the buildings that come form the same blueprint will have many of the same features (same number of rooms, same structure, same shape, etc.). This is also true in coding. Classes are the category of design or function while instances/objects (below) are the the actual specific buildings that arise from the blueprints.
  • Classes are like neatly packaged boxes. They contain valuable information for different categories of programs.
  • When using classes. Usually the class information is coded in one file and the call is coded in another file.

Objects

  • As described above objects are specific instances of the blueprint of a class. They are functions, variables, or a data structure that fits into one of the classes. Each object will have all the qualities that are defined by its class but also some more specific qualities that make it unique.

Instance Methods

  • Are functions that have self for the first value. They are associated with a specific object.

Instance Variables

  • Are variables that are associated with all instances in a class. They are like class variables.

Standard Library and importing data

  • Python has a standard library of classes. Other classes can be used from other outside sources by the use of import.

Module

  • is a segment of code that logically fits together. Modules can define functions and can be imported to use at a later time

OOP in CSS and HTML

  • The principles OOP are used in web development as well. CSS, for example, provides code modules which can be reused over and over again throughout the website. This article provides some great examples of using OOP principles for CSS coding. This approach removes all unnecessary repetition.
  • Unfortunately HTML is not an object oriented language so there is not effective means of using object oriented methods with it. This is why CSS is so valuable to avoiding repetition.

Lesson 10: JavaScript

  • JavaScript can be used to give websites “life”. It manipulates HTML and CSS giving websites functionality and less of a static feel.
  • Java is added to an html document by text such as the following:
    •      <script scr=“js/jQuery.js”></script>
  • Vanilla is normal JavaScript. jQuery is a special pre-made JavaScript library.
  • JavaScript can be experimented with in console section of the developer tools in most browsers.
  • console.log(“string”) is like the print feature for JavaScript
  • variables in JavaScript function a lot like variables in Python, but they require a “var” keyword.
  • Like Python, JavaScript does math. One of the neat math features JavaScript can do is called modulus (computed with a “%”) it returns the remainder from the division of two numbers.
